Backbone - A Refresher

I haven’t been actively using Backbone for a couple of months now.

The Nature of Backbone

Out of the box Backbone is an unopinionated library for bringing structure to an application. It provides you with data-modelling, bare-bone views and a router. That’s probably all you’re going to need for a small and simple application.

It’s unopinionated because it’s intended to be used as a foundation library rather than a framework.

Use Promises

Even if jQuery’s deferred objects aren’t A+ they’re still extremely useful in Backbone.

model.save().then(success, failure)

There’s nothing stopping you from adopting a different promise library. The point I wish to make is that it’s easy to get sucked into using events like ‘sync’ or callbacks and not seeing the light of day for a while.
