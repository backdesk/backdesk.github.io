Backbone - MV ... What?

Are Backbone’s Views … Controllers?

It’s tricky to find a ‘one size fits all’ definition of a controller and its responsibilities. Responsibilities are open to interpretation based on the platform [1] and framework you’re using. I think that the underlying principle rests at separating concerns. How a code base achieves this separation is what ultimately defines the role of a controller.

In a bare bone Backbone application ‘Backbone.View’ could be considered a controller as its logic drives a ‘dumb view’ aspect using templates. Model events and user interactions are handled within the view logic and reflected in the rendered template.
